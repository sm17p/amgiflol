# Issue management

**Source of truth:** [GitHub issues](https://github.com/sm17p/amgiflol/issues) — context, refinements, labels, and tracking live in the issue body and comments.

Use the repo issues for:

- Refined descriptions and acceptance criteria (in the issue body)
- Labels for filtering and triage
- Discussion and tracking (comments)

## Labels

| Label              | Use for                                |
| ------------------ | -------------------------------------- |
| `bug`              | Something broken or incorrect behavior |
| `documentation`    | Docs, README, FAQ, browser support     |
| `enhancement`      | New feature or improvement             |
| `good first issue` | Clear, scoped tasks for contributors   |
| `question`         | Open questions, naming, investigation  |
| `UI/UX`            | User-facing behavior, flows, layout    |
| `Brainstorm`       | Ideas, naming, long-running discussion |
| `infrastructure`   | CI/CD, release, monitoring, tooling    |

To add context, research, or acceptance criteria to an issue, edit the issue body or add a comment on the issue.
